## 2026-06-11 修复前端无法登录问题

**问题根因**：vite.config.js 中未配置 proxy，前端 /api 请求无法转发到后端 8000 端口，导致登录时请求发到 5173 端口返回 404。

**修复**：在 vite.config.js 的 server 配置中添加 proxy：`'/api': { target: 'http://127.0.0.1:8000', changeOrigin: true }`

**额外发现**：后端登录路径为 `/api/auth/login-json`（JSON 格式），前端 api/auth.js 中已正确使用该路径。后端也有 `/api/auth/login`（form 格式）。

**端口管理**：Git Bash 下 taskkill 可能杀不干净 node 进程，需用 PowerShell 的 Get-NetTCPConnection + Stop-Process 按端口杀进程。
