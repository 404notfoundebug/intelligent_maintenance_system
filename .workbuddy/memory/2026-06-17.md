## 2026-06-17 修复认证 Bug（关键修复）

### 根因
前端大量页面使用裸 `axios.get/post/put/delete` 而非封装的 `request` 实例，导致：
- **请求拦截器不生效** → 不会自动添加 `Authorization: Bearer <token>` header
- **响应拦截器不生效** → 不会自动解包 `data` 字段
- 后端 OAuth2PasswordBearer 收不到 token，返回 401 + `"Not authenticated"`

### 修复内容
1. **`frontend/src/api/request.js`**：修复 401 处理中先 `removeToken()` 再读 `role` 的顺序 bug
2. **`backend/app/utils/response.py`**：统一 `code: 0` → `code: 200`
3. **批量替换**：所有 19 个 Vue 页面 + 组件中的 `axios.xxx()` → `request.xxx()`，同时适配拦截器解包后的数据访问方式（`res.data.data` → `res`）
4. 修复的文件：AdminStats, AuditLog, DataBackup, KnowledgeGraph, RoleConfig, UserManagement, DeviceForm, InspectionTemplateForm, KnowledgeDetail, KnowledgeUpload, MaintenanceAuditDetail, WorkerCaseReport, WorkerDeviceView, WorkerFaultReport, WorkerInspection, WorkerKnowledgeSearch, WorkerMaintenance, WorkerProfile, WorkerRepairAdvice

- 用户名：`admin`，密码：`admin123456`（见 `init_db.py`）

- `vite build` 通过，零错误

## 2026-06-17 修复工人端设备信息查看和维保记录加载问题

### 问题
工人端 WorkerDeviceView 和 WorkerMaintenance 内容一直加载不出来，无法与管理员端同步。

### 根因分析（5 个 Bug）

**WorkerDeviceView.vue：**
1. **表格字段名不匹配**：前端用 `name`/`type`/`location`，后端返回 `device_name`/`device_type`/`installation_location` → 表格列全空
2. **状态判断错误**：`row.status === '正常'` 永远为 false，后端返回英文值 `"normal"`
3. **数据未解包**：`devices.value = res || []`，但 `res` 是 `{ total, page, page_size, items }` 分页对象，应取 `res.items`

**WorkerMaintenance.vue：**
4. **数据未解包**：`records.value = r1 || []`，`r1` 是分页对象，应取 `r1.items`
5. **字段名不匹配 + 路由不存在**：表格列 `content` 在后端不存在（后端是 `conclusion`）；新建维保记录 POST `/api/maintenance/records` 后端无此路由（维保记录只能从点检工单生成 `POST /api/maintenance/records/from-order/{order_id}`）

### 修复内容
- **WorkerDeviceView.vue**：修正所有字段名为后端一致；状态用 statusOptions 映射英文→中文+颜色；取 `res.items`；增加分页控件（el-pagination）
- **WorkerMaintenance.vue**：取 `r1.items`；字段 `content`→`conclusion`；新建改为"生成维保记录"Dialog，选择已完成点检工单后调用 `from-order` 接口；增加分页控件；报告预览增加复制功能
